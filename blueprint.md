# 프로젝트 청사진: 행운의 바퀴

## 1. 개요

**행운의 바퀴**는 사용자가 일상적인 상황에서 어렵고 사소한 결정을 내릴 때 재미있고 공정한 방법을 제공하는 무료 온라인 랜덤 룰렛 웹 애플리케이션입니다. 이 서비스는 점심 메뉴 선택, 이벤트 당첨자 추첨, 게임 벌칙 결정 등 다양한 상황에서 의사결정을 도와주는 도구입니다.

이 프로젝트의 주요 목표는 사용자에게 직관적이고 즐거운 경험을 제공하고, 구글 애드센스 프로그램을 통해 수익을 창출하는 것입니다. 이를 위해 사이트는 유용한 기능, 풍부한 정보, 그리고 매력적인 디자인을 갖추어야 합니다.

## 2. 디자인 및 기능 명세

### 2.1. 시각 디자인 (Aesthetics)

- **테마**: 활기찬 카니발, 축제 분위기
- **색상 팔레트**: 밝고 선명한 원색(빨강, 노랑, 파랑 등)을 사용하여 즐거운 느낌을 강조합니다. 그라데이션과 부드러운 그림자 효과로 깊이감을 더합니다.
- **타이포그래피**: 제목에는 'Black Han Sans' 또는 'Jua'와 같은 굵고 둥근 글꼴을 사용하여 주목도를 높이고, 본문에는 가독성 좋은 고딕 계열 글꼴을 사용합니다.
- **아이콘 및 이미지**: 룰렛, 별, 전구, 음식, 선물 등 상황에 맞는 다채로운 아이콘을 사용하여 시각적 이해를 돕습니다.
- **애니메이션**: 룰렛이 돌아가는 동안 부드러운 회전 애니메이션과 멈춘 후 축하하는 의미의 색종이(Confetti) 효과를 적용하여 동적인 즐거움을 제공합니다.

### 2.2. 핵심 기능

- **랜덤 룰렛**:
    - 사용자가 지정한 항목들로 구성된 원형 룰렛을 화면 중앙에 표시합니다.
    - '돌리기!' 버튼을 누르면 룰렛이 회전하다가 서서히 멈추며 하나의 항목을 가리킵니다.
    - 결과는 팝업이나 별도의 영역에 강조하여 표시됩니다.
- **사용자 커스터마이징**:
    - **항목 수 조절**: 2개에서 12개까지 룰렛의 칸 수를 조절할 수 있습니다.
    - **항목 내용 편집**: 각 칸에 들어갈 텍스트를 사용자가 직접 입력할 수 있습니다.
    - **프리셋**: '점심 메뉴', '예/아니오' 등 자주 사용되는 항목들을 미리 만들어두어 사용자가 쉽게 불러올 수 있도록 합니다.
- **사운드 효과**: 룰렛이 돌아갈 때 '째깍째깍'하는 소리와 결과가 나왔을 때 '팡파르' 효과음을 넣어 몰입감을 높입니다.

### 2.3. 페이지 구성

- **메인 페이지 (`index.html`)**:
    - 룰렛과 설정 패널이 있는 핵심 기능 페이지.
    - 서비스 소개, 사용 방법, 활용 예시, FAQ 등 풍부한 정보를 제공하여 페이지의 가치를 높입니다.
- **소개 페이지 (`about.html`)**:
    - 서비스의 목적, 주요 기능, 기술 정보, 연락처 등을 상세히 설명합니다.
- **가이드 페이지 (`guide.html`)**:
    - '공정하고 재미있는 결정을 내리는 5가지 창의적인 방법'과 같은 주제의 독창적인 콘텐츠를 제공하여 사이트의 전문성과 가치를 높입니다.
- **개인정보처리방침 (`privacy.html`)** 및 **이용약관 (`terms.html`)**:
    - 법적 고지사항을 명확하고 상세하게 기술합니다. 수집하는 데이터의 종류와 목적을 구체적으로 명시하여 신뢰도를 높입니다.

## 3. 애드센스 승인 전략 (현재 진행 계획)

애드센스 승인 거절 사유인 '가치 없는 콘텐츠' 문제를 해결하기 위해 다음 단계를 수행합니다.

1.  **`blueprint.md` 파일 생성**: 프로젝트의 목적과 계획을 명확히 문서화합니다. (완료)
2.  **신규 콘텐츠 페이지 추가**:
    - `guide.html` 파일을 새로 생성합니다.
    - '공정하고 재미있는 결정을 내리는 방법'과 관련된 유용하고 독창적인 기사를 작성합니다. 이 페이지는 단순한 정보 나열이 아닌, 사용자에게 실제적인 도움을 주는 고품질 콘텐츠로 구성합니다.
3.  **기존 콘텐츠 강화 및 탐색 개선**:
    - 모든 페이지(`index.html`, `about.html`, `privacy.html`, `terms.html`, `guide.html`)의 푸터에 `guide.html`로 연결되는 탐색 링크를 추가합니다.
    - `index.html`에 `guide.html` 페이지를 소개하는 섹션을 추가하여 사용자의 방문을 유도합니다.
    - `privacy.html`과 `terms.html`에 수집되는 비개인 식별 정보의 종류와 사용 목적을 더 구체적으로 명시하여 콘텐츠의 깊이를 더합니다.
4.  **Firebase MCP 설정**:
    - `.idx/mcp.json` 파일을 생성하여 Firebase 관련 서버 설정을 추가합니다.
5.  **검토 및 제출**:
    - 모든 변경 사항이 사이트에 올바르게 반영되었는지 확인한 후, 구글 애드센스에 재검토를 요청합니다.
